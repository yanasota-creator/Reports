# レポート作成スタイルガイド

工学実験レポートをMarkdown（GitHub）で作成する際のルール。次回以降のレポートはこれに従って作成する。

## 1. 句読点

- 読点・句点は「、」「。」を使う（「，」「．」は使わない）

## 2. 太字（`**...**`）の使い方

- **OK**：見出し、表・図のキャプション（例：`**表5.1　○○**`）のみ
- **NG**：本文中で一部の語句・フレーズを強調するための太字（例：`〜が**重要である**．`のような使い方はしない）、箇条書きの項目名ラベルを太字にすること（例：`**直交ゲージ**：〜`のような使い方はしない）

## 3. 数式の書き方

- **文中に埋め込む小さい数式**（`$E=200$MPa`など）：前後にスペースを入れず詰めて書く
  - 例：`縦弾性係数は$E_{実験}=221{,}272$MPaであった。`
- **1行（段落）を使う大きいブロック数式**（`$$ ... $$`）：前後に空行を入れて独立した段落にする（スペースの詰め込みは不要）

## 4. 表・図のキャプション

- 表には**表X.X　タイトル**、図には**図X　タイトル**のキャプションを付ける（本文中で「図1」「表5.2」のように参照している番号と一致させる）
- 表のキャプションは表の**上**、図のキャプションは図（画像）の**下**に置く
- 図（画像）は、対応する本文（説明文）の**直下**に配置する
- 表の列の配置：**数値の列は右寄せ**（`---:`）、**文字列（項目名等）の列は左寄せ**（`:---`）、**見出し的に短い記号だけの列は中央寄せ**（`:---:`）でも可。数値列は右寄せに統一する

## 5. 参考文献

- `[1]`、`[2]`...の各項目は、**間に空行を1つ入れて**改行する（空行が無いとGitHub上で1行に繋がって表示されてしまう）
- 書式：`[番号] 出典名．「タイトル」．URL　参照日:YYYY/M/D`

```
[1] 出典名．「タイトル」．https://example.com　参照日:2026/6/28

[2] 出典名．「タイトル」．https://example.com　参照日:2026/6/28
```

## 6. ファイル名・フォルダ構成

- ファイル名は**英数字とアンダースコアのみ**（日本語・スペースは使わない）
  - 日本語ファイル名はURLが極端に長くなり読み込みエラーの原因になる
  - スペースはMarkdown PDFが画像を読み込めなくなる原因になる（`Stress-strain diagram.png` → NG、`Stress-strain_diagram.png` → OK）
- 構成：

```
report.md / report_xxx.md   ← リポジトリ直下
data/                        ← 測定データ（csv等）
images/                      ← グラフ・図（png推奨、pdfはMarkdown上で表示されないため不可）
```

## 7. レポートの基本構成

```
1. 目的
2. 使用機器
3. 実験原理
4. 実験手順
5. 実験結果（表・図を含む）
6. 考察
（必要な場合のみ）IV. 追加調査課題
参考文献
```

## 8. 考察を書く上での姿勢

- データに異常値がある場合、消さずにそのまま記載し、考察で原因を推測する（操作ミス・読み取りミスなど、具体的な状況がわかっていればそれを書く）
- 文献値との比較を行う場合は、誤差(%)を計算して明記し、出典を参考文献に記載する

## 9. グラフ・図の描き方

- データ点は大きく、はっきりと描く
- データ点同士を単純な折れ線で結ばない。最小二乗法などによる回帰直線・回帰曲線のように、根拠のある線を引く
- 軸には必ず物理量と単位を明記する（例：`コイル電流 I [A]`）
- 目盛りの数値は読み取りやすい間隔にし、目盛り線（グリッド線）を入れる
- グラフには必ずタイトルをつける
- 白黒印刷を想定し、複数系列を1つのグラフに重ねる場合は、色だけでなく線種（実線・破線・点線・一点鎖線）とマーカー形状（○・□・△・×など）を組み合わせて区別できるようにする

## 10. 参考文献の本文中引用

- 本文中で参考文献を引用する箇所は、上付き文字で書く
- Markdownでは`<sup>[1]</sup>`のようにHTMLタグを使う
